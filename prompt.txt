For Phase 1 (Company User Side), don't start with Products or Suppliers first.

Build in the order that removes dependencies and lets you test the system properly.

Step 1: Authentication & Authorization

Build:

Company Login (/login)
Auth.js credentials login
Session management
Role-based middleware
Protected routes

Roles:

ADMIN
MANAGER
STAFF

After login:

/dashboard
Step 2: Dashboard Layout & Navigation

Build:

Sidebar
Header
User profile dropdown
Mobile navigation
Dashboard home page

Create empty pages:

/dashboard
/dashboard/products
/dashboard/suppliers
/dashboard/purchase-orders
/dashboard/sales-orders
/dashboard/transfers
/dashboard/history
/dashboard/reports
/dashboard/users
/dashboard/settings

Don't implement functionality yet.

Step 3: Company Settings

Build:

/dashboard/settings

Show:

Company Name
Company Code
Logo
Phone
Address

This verifies your company data is loading correctly.

Step 4: Brand Management

Build:

/dashboard/brands

CRUD:

Create Brand
Update Brand
Delete Brand
Search Brand

Examples:

Crocs
Nike
Adidas
Step 5: Category Management

Build:

/dashboard/categories

Examples:

Sandals
Sports Shoes
Slippers
Casual Shoes
Step 6: Supplier Management

Build:

/dashboard/suppliers

CRUD:

Name
Phone
Email
Address
Step 7: Product Management

Now you have:

Brand
Category
Supplier

Build:

/dashboard/products

Features:

Product CRUD
Variant CRUD
Search
Filters

Example:

Crocs Classic
 ├─ Size 7 Black
 ├─ Size 8 Black
 └─ Size 9 Blue
Step 8: Inventory Movement Engine

Create:

InventoryMovement

This is the heart of your system.

Track:

PURCHASE
SALE
TRANSFER_IN
TRANSFER_OUT
ADJUSTMENT

Even before Purchase Orders.

Step 9: Purchase Orders

Build:

/dashboard/purchase-orders

Features:

Create PO
Receive stock
Partial receiving
Auto InventoryMovement entries
Step 10: Sales Orders

Build:

/dashboard/sales-orders

Features:

Create Invoice
Reduce stock
Record movement
Step 11: Stock Transfers

Build:

/dashboard/transfers

Move stock:

SHOP → GODOWN
GODOWN → SHOP
Step 12: Reports

Build:

/dashboard/reports
Stock Report
Sales Report
Purchase Report
Low Stock Report
Step 13: User Management

Build last:

/dashboard/users

Because by now you'll know exactly what permissions each role needs.

Final Build Order
1. Authentication
2. Dashboard Layout
3. Company Settings
4. Brands
5. Categories
6. Suppliers
7. Products
8. Inventory Movement
9. Purchase Orders
10. Sales Orders
11. Stock Transfers
12. Reports
13. User Management

If I were building your footwear inventory SaaS from scratch, this is the exact order I would follow. It minimizes rework and keeps dependencies clean.
